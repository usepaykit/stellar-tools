---
title: 'Credits'
description: 'Manage customer credit balances and usage for metered billing'
---

Credits power metered billing: customers have a balance per metered product. You deduct when they use the service, refund to undo a charge, or grant more credits. Amounts are sent in raw units (e.g. tokens, bytes); the API converts to credits using the product's `unitDivisor` and `unitsPerCredit`. All endpoints require `x-api-key`.

## Get credit balance

Returns the current credit balance for a customer and metered product (balance, consumed, granted). Response `200`: `{ "data": { "id", "customerId", "productId", "balance", "consumed", "granted", "metadata", "createdAt", "updatedAt", "environment", "organizationId" } }`

<CodeGroup>

```typescript SDK
// Balance is included when consuming or checking. For balance-only, use REST:
const res = await fetch(
  "https://stellartools.dev/api/customers/cust_xxx/credits/prod_xxx",
  { headers: { "x-api-key": process.env.STELLAR_TOOLS_API_KEY! } }
);
const { data } = await res.json();
```

```bash cURL
curl -X GET "https://stellartools.dev/api/customers/cust_xxx/credits/prod_xxx" \
  -H "x-api-key: your_api_key"
```

</CodeGroup>

## List credit transactions

Returns the transaction history for a customer's credits. Optionally filter by product and paginate. Response `200`: `{ "data": [ { "id", "customerId", "productId", "balanceId", "amount", "balanceBefore", "balanceAfter", "type", "reason", "metadata", "createdAt", ... }, ... ] }`. Type is `deduct`, `refund`, or `grant`.

| Query    | Type   | Default | Description                          |
|----------|--------|--------|--------------------------------------|
| productId | string | —      | Filter by metered product ID.        |
| limit   | number | 50     | Max number of transactions to return. |
| offset  | number | 0      | Number of transactions to skip.      |

<CodeGroup>

```typescript SDK
const transactions = await stellar.credits.getTransactions("cust_xxx", {
  productId: "prod_xxx",
  limit: 50,
  offset: 0,
});
```

```bash cURL
curl -X GET "https://stellartools.dev/api/customers/cust_xxx/credits/transactions?productId=prod_xxx&limit=50&offset=0" \
  -H "x-api-key: your_api_key"
```

</CodeGroup>

## Post credit transaction

Deduct, refund, or grant credits for a customer on a metered product. Amount is in raw units (e.g. tokens, bytes); the server converts to credits using the product's `unitDivisor` and `unitsPerCredit`. Response `200`: `{ "isSufficient": true, "transaction": { "id", "amount", "balanceBefore", "balanceAfter", "type", ... }, "balance": { updated balance object } }`. For `dryRun: true`, `transaction` is null and `balance` is the current balance (unchanged). Errors: `400` for validation or invalid product/customer; `400` with message "Insufficient credits" when deduct would exceed balance; `400` with message "Invalid Meter Configuration" when no credit balance exists.

| Body field | Type    | Required | Description |
|------------|---------|----------|-------------|
| amount    | number  | Yes      | Raw usage amount (e.g. 250 tokens). Converted to credits by the product's conversion rules. |
| type      | string  | Yes      | `deduct` (charge), `refund` (add back), `grant` (e.g. top-up). |
| reason    | string  | No       | Optional description. |
| metadata  | object  | No       | Optional key-value data. |
| dryRun    | boolean | No       | If true, only check sufficiency for deduct. No balance or transaction is written. |

<CodeGroup>

```typescript SDK
// Deduct (consume) credits
const result = await stellar.credits.consume("cust_xxx", {
  productId: "prod_xxx",
  rawAmount: 250,
  reason: "AI completion tokens",
  metadata: { requestId: "req_123" },
});

// Refund credits
await stellar.credits.refund("cust_xxx", {
  productId: "prod_xxx",
  amount: 250,
  type: "refund",
  reason: "Reversed usage",
});

// Check sufficiency (dry run)
const check = await stellar.credits.check("cust_xxx", {
  productId: "prod_xxx",
  rawAmount: 100,
  dryRun: true,
});
```

```bash cURL
curl -X POST "https://stellartools.dev/api/customers/cust_xxx/credits/prod_xxx/transaction" \
  -H "x-api-key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 250,
    "type": "deduct",
    "reason": "AI completion tokens",
    "metadata": { "requestId": "req_123" },
    "dryRun": false
  }'
```

</CodeGroup>
