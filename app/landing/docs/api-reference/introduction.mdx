## Base URL

```
https://your-domain.com/api
```

For local development: `http://localhost:3000/api`

## Authentication

Authenticate using one of the following headers:

| Header | Use case |
|--------|----------|
| `x-api-key` | Server-side or dashboard API key |
| `x-session-token` | User session (e.g. dashboard logged-in user) |

All endpoints that require auth return `400` with `{ "error": "API key or session token is required" }` when missing.

<Note>
  Get your API key from the [Dashboard](https://docs.stellartools.dev) → Settings → API Keys.
</Note>

## API resources

| Resource | Description |
|----------|-------------|
| [Checkout](/docs/api-reference/checkout) | Create and manage payment checkouts (one-time or subscription) |
| [Customers](/docs/api-reference/customers) | Create and list customers |
| [Webhooks](/docs/api-reference/webhooks) | Create, update, and delete webhook endpoints |
| [Subscriptions](/docs/api-reference/subscriptions) | Create and manage subscriptions |
| [Payments](/docs/api-reference/payments) | Retrieve payment status |
| [Refunds](/docs/api-reference/refunds) | Create refunds for payments |
| [Products](/docs/api-reference/products) | Create products (one-time or subscription) |
| [Payout](/docs/api-reference/payout) | Send payouts to a Stellar address (session only) |

## Response format

Success responses wrap data in a `data` field:

```json
{
  "data": { ... }
}
```

Errors return a JSON object with an `error` field and an appropriate HTTP status (4xx/5xx).

## Rate limits

Rate limits depend on your plan. Check the dashboard for current limits.
