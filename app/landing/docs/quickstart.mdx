Get your first checkout working with the Stellar Tools API.

## Prerequisites

- An account on [Stellar Tools](https://usepaykit.com) (or your deployed dashboard).
- An API key from the dashboard (Settings → API Keys).
- Node.js 18+ (or any HTTP client) for the examples below.

## Step 1: Create a product (optional)

For a **product-based checkout**, create a product first. For a **direct amount** checkout (e.g. donation), skip to Step 2.

```bash
curl -X POST https://your-api.com/api/product \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Pro Plan",
    "description": "Monthly access",
    "type": "subscription",
    "assetId": "YOUR_ASSET_ID",
    "priceAmount": 10000000,
    "recurringPeriod": "monthly"
  }'
```

Use the returned `id` (e.g. `prod_xxx`) as `productId` when creating a checkout.

## Step 2: Create a checkout

**Product checkout** (use the product ID from Step 1):

```bash
curl -X POST https://your-api.com/api/checkout \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "productId": "prod_xxx",
    "customerEmail": "buyer@example.com",
    "successUrl": "https://yoursite.com/success"
  }'
```

**Direct amount checkout** (no product):

```bash
curl -X POST https://your-api.com/api/checkout \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 10.5,
    "assetCode": "XLM",
    "customerEmail": "buyer@example.com",
    "successUrl": "https://yoursite.com/thanks"
  }'
```

The response includes `data.paymentUrl` — send your customer there to pay, or use `data.id` with `GET /api/checkout/:id/uri?environment=mainnet` to get a Stellar payment URI for wallet redirect or QR.

## Step 3: Poll for payment status (optional)

Your payment page or client can poll checkout status (no API key; CORS enabled):

```bash
curl "https://your-api.com/api/checkout/CHECKOUT_ID/status"
```

Response: `{ "status": "pending" | "completed" | "expired" | "failed" }`
