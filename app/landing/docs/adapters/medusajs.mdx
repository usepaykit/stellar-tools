title: "MedusaJS Adapter"

---

The MedusaJS adapter registers Stellar as a payment provider in [Medusa v2](https://medusajs.com). Customers pay with Stellar (XLM or other assets); you get webhooks and full payment lifecycle.

## Install

```bash
pnpm add @stellartools/medusajs-adapter
```

## Configure

Register the payment module and pass your Stellar Tools API key. The adapter exports `StellarToolsMedusaAdapter` and is registered via `ModuleProvider(Modules.PAYMENT, { services: [StellarToolsMedusaAdapter] })`:

```ts
import { defineConfig, loadEnv } from "@medusajs/framework/utils";
import { StellarTools } from "@stellartools/medusajs-adapter";

loadEnv(process.env.NODE_ENV || "development", process.cwd());

module.exports = defineConfig({
  projectConfig: {
    databaseUrl: process.env.DATABASE_URL,
  },
  modules: [
    {
      resolve: "@medusajs/payment",
      options: {
        providers: [
          {
            resolve: "@stellartools/medusajs-adapter",
            id: "stellar",
            options: {
              apiKey: process.env.STELLAR_TOOLS_API_KEY,
              webhookSecret: process.env.STELLAR_TOOLS_WEBHOOK_SECRET,
              debug: true,
            },
          },
        ],
      },
    },
  ],
});
```

## Options

| Option          | Type      | Description                                               |
| --------------- | --------- | --------------------------------------------------------- |
| `apiKey`        | `string`  | Stellar Tools API key (required).                         |
| `debug`         | `boolean` | Enable debug logging (default: true).                     |
| `webhookSecret` | `string`  | Webhook secret for verifying Stellar webhooks (required). |

## Behavior

- Payment provider identifier: `stellar`.
- Initiate payment: creates a [direct checkout](/api-reference/checkout) with `amount` and `currency_code`; returns `payment_url` for redirect.
- Get / authorize / retrieve payment: uses [payments API](/api-reference/payments); status is mapped to Medusa session status (`REQUIRES_MORE`, `AUTHORIZED`, `ERROR`).
- Capture: no-op (treats as captured).
- Refund: calls [refunds API](/api-reference/refunds); requires `receiverWalletAddress` and optional `reason` in payment data.
- Account holder: create/update/delete map to Stellar [customers](/api-reference/customers).
- Webhooks: handles `payment.pending`, `payment.confirmed`, `payment.failed` and forwards to Medusa.

Blockchain payments are immutable: cancel, delete, and update payment throw `NOT_ALLOWED`.

## Webhook setup

Configure your [Stellar Tools webhook](/api-reference/webhooks) to post to Medusa’s provider webhook URL. The adapter parses the payload and returns the appropriate `PaymentActions` (e.g. `SUCCESSFUL` for `payment.confirmed`).

## Example: Initiate payment

When a customer checks out, Medusa calls the provider’s `initiatePayment` with `amount`, `currency_code`, and context. The adapter creates a direct checkout and returns:

- `id`: checkout ID
- `status`: `REQUIRES_MORE`
- `data.payment_url`: URL for the customer to complete payment with Stellar

Redirect the customer to `payment_url`; after payment, your webhook will receive the event and Medusa will update the order.
