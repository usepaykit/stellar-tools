## Base URL

```
https://stellartools.dev/api
```

For local development: `http://localhost:3000/api`

## Authentication

Send your API key in the `x-api-key` header. If auth is missing, endpoints return `400` with `{ "error": "API key or session token is required" }`.

<Note>Get your API key from the StellarTools dashboard (Settings → API Keys).</Note>

## API resources

| Resource                                      | Description                                                    |
| --------------------------------------------- | -------------------------------------------------------------- |
| [Checkout](/api-reference/checkout)           | Create and manage payment checkouts (one-time or subscription) |
| [Customers](/api-reference/customers)         | Create, list, get, update, and delete customers                |
| [Webhooks](/api-reference/webhooks)           | Create, update, and delete webhook endpoints                   |
| [Subscriptions](/api-reference/subscriptions) | Create and manage subscriptions (pause, resume, cancel)        |
| [Payments](/api-reference/payments)           | Retrieve payment status                                        |
| [Refunds](/api-reference/refunds)             | Create refunds for payments                                    |
| [Products](/api-reference/products)           | Create products (one-time, subscription, or metered)           |
| [Credits](/api-reference/credits)             | Manage credit balances and usage for metered billing           |

## Response format

Success responses wrap data in a `data` field:

```json
{
  "data": { ... }
}
```

Errors return a JSON object with an `error` field and an appropriate HTTP status (4xx/5xx).

## Rate limits

Rate limits depend on your plan. Check the dashboard for current limits.
