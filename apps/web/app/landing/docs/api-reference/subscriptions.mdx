Subscriptions are recurring payments tied to a product of type `subscription` and customer wallets.

## Create subscriptions (bulk)

Creates subscriptions for multiple customers on a subscription product. Each customer must have a Stellar wallet. Response `200`: `{ "data": { "count": 2, "failed": [] } }`. The `failed` array lists any customers that could not be subscribed (e.g. missing wallet).

<CodeGroup>

```typescript SDK
const result = await stellar.subscriptions.create({
  productId: "prod_sub_xxx",
  customerIds: ["cust_1", "cust_2"],
  period: {
    from: "2025-01-01T00:00:00.000Z",
    to: "2025-02-01T00:00:00.000Z",
  },
  cancelAtPeriodEnd: false,
});
```

```bash cURL
curl -X POST "https://stellartools.dev/api/subscriptions" \
  -H "x-api-key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "productId": "prod_sub_xxx",
    "customerIds": ["cust_1", "cust_2"],
    "period": {
      "from": "2025-01-01T00:00:00.000Z",
      "to": "2025-02-01T00:00:00.000Z"
    },
    "cancelAtPeriodEnd": false
  }'
```

</CodeGroup>

## List subscriptions

Response `200`: `{ "data": [ { subscription objects } ] }`

<CodeGroup>

```typescript SDK
const subscriptions = await stellar.subscriptions.list("cust_xxx");
```

```bash cURL
curl -X GET "https://stellartools.dev/api/subscriptions?customerId=cust_xxx" \
  -H "x-api-key: your_api_key"
```

</CodeGroup>

## Get subscription

Response `200`: `{ "data": { subscription object } }`

<CodeGroup>

```typescript SDK
const subscription = await stellar.subscriptions.retrieve("sub_xxx");
```

```bash cURL
curl -X GET "https://stellartools.dev/api/subscriptions/sub_xxx" \
  -H "x-api-key: your_api_key"
```

</CodeGroup>

## Update subscription

Update metadata or set `cancelAtPeriodEnd`. Response `200`: `{ "data": { updated subscription } }`

<CodeGroup>

```typescript SDK
const updated = await stellar.subscriptions.update("sub_xxx", {
  metadata: { plan: "pro" },
  cancelAtPeriodEnd: true,
});
```

```bash cURL
curl -X PUT "https://stellartools.dev/api/subscriptions/sub_xxx" \
  -H "x-api-key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": { "plan": "pro" },
    "cancelAtPeriodEnd": true
  }'
```

</CodeGroup>

## Pause subscription

Pauses the subscription on-chain. The REST API may require `customerAddress` (Stellar public key) and `productId` in the body. Response `200`: `{ "data": { subscription } }`

<CodeGroup>

```typescript SDK
const subscription = await stellar.subscriptions.pause("sub_xxx");
```

```bash cURL
curl -X POST "https://stellartools.dev/api/subscriptions/sub_xxx/pause" \
  -H "x-api-key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "customerAddress": "G...",
    "productId": "prod_xxx"
  }'
```

</CodeGroup>

---

## Resume subscription

Resumes a paused subscription. REST body must include `customerAddress` and `productId`. Response `200`: `{ "data": { updated subscription } }`

<CodeGroup>

```typescript SDK
const subscription = await stellar.subscriptions.resume("sub_xxx");
```

```bash cURL
curl -X POST "https://stellartools.dev/api/subscriptions/sub_xxx/resume" \
  -H "x-api-key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "customerAddress": "G...",
    "productId": "prod_xxx"
  }'
```

</CodeGroup>

## Cancel subscription

<CodeGroup>

```typescript SDK
const subscription = await stellar.subscriptions.cancel("sub_xxx");
```

```bash cURL
curl -X POST "https://stellartools.dev/api/subscriptions/sub_xxx/cancel" \
  -H "x-api-key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "customerAddress": "G...",
    "productId": "prod_xxx"
  }'
```

</CodeGroup>
