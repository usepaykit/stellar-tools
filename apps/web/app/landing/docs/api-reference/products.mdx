Products define what you sell: a one-time payment, a subscription with a price and asset, or a metered product where customers pay by usage (credits).

## Create product

Type: `one_time`, `subscription`, or `metered`. `assetId` is the ID of the asset (e.g. XLM, USDC) for pricing. `priceAmount` is the price in the asset's smallest unit (e.g. stroops for XLM). For subscriptions, `recurringPeriod` can be `day`, `week`, `month`, or `year`. Optional: `unit`, `unitDivisor`, `unitsPerCredit`, `creditsGranted`, `creditExpiryDays`, `metadata`. You can also send `multipart/form-data` with image files. Response `200`: `{ "data": { "id", "name", "type", "priceAmount", "status", "assetId", "recurringPeriod", ... } }`

<CodeGroup>

```typescript SDK
// Product creation is via REST or dashboard. Use fetch with your API key:
const res = await fetch("https://stellartools.dev/api/product", {
  method: "POST",
  headers: {
    "x-api-key": process.env.STELLAR_TOOLS_API_KEY!,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Pro Plan",
    description: "Monthly access",
    type: "subscription",
    assetId: "asset_xxx",
    priceAmount: 10,
    recurringPeriod: "month",
    metadata: {},
  }),
});
const { data } = await res.json();

// Retrieve a product with the SDK
const product = await stellar.products.retrieve("prod_xxx");
```

```bash cURL
curl -X POST "https://stellartools.dev/api/product" \
  -H "x-api-key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Pro Plan",
    "description": "Monthly access",
    "type": "subscription",
    "assetId": "asset_xxx",
    "priceAmount": 10,
    "recurringPeriod": "month",
    "unit": "credits",
    "unitDivisor": 1,
    "unitsPerCredit": 1,
    "creditsGranted": 100,
    "metadata": {}
  }'
```

## Metered products

For `type: "metered"`, customers are charged by usage via credits. Set: `unit` (e.g. tokens, MB, API calls), `unitDivisor` (e.g. 1 or 1_000_000 for bytes), `unitsPerCredit` (how many units = 1 credit), `creditsGranted` (initial credits), and optionally `creditExpiryDays`. Credits are then managed with the [Credits API](/api-reference/credits). See [Metered billing](/metered-billing) for the full flow and examples.
